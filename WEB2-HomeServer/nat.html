NAT (Network Address Translation)

NAT 의 기술을 통해서 private ip address 를 가지고 있는 장치들이 외부에 (public ip address, 즉 WAN) 존재하는 장치들과 통신을 할 수 있는 것이다.

Client 측면에서 NAT 의 작동 방식

만약에 private ip address 즉 LAN 에 포함되어진 주소 여기에서는 192.168.0.4 가 외부로 부터, 즉 여기에서는 public ip address 또는 WAN 으로 부터 어떠한 정보를 요청하게 되면 그 신호는 먼저 Gateway Address 인 192.168.0.1 에 들어가게 된다. 그리고 나서 공유기는 LAN 에 원하는 ip address 가 없다는 것을 인지하고 외부로 나가서 요청되어진 데이터를 찾을려고 할 것이다. 하지만 여기에서 문제점이 발생하게 된다. 이는 바로 private ip address 오직 LAN 에서만 사용이 가능 하다는 것이다. 그렇다면 사설 ip address 가 public ip address 랑 통신을 나누기 위해서는 어떠한 기술이 필요한 것일까? 이는 바로 NAT (Network Address Translation) 이라는 기술이다. 이 기술은 2가지 정보를 기역하고 있다. 먼저 누가 이 요청을 보냈는 가를 router 에 저장을 하게 된다, 즉 여기에서는 192.168.0.4 가 될것이다. 두번째로 192.168.0.4 이라는 ip address 대신에 59.6.66.238 이라는 ip address 를 가지고 외부랑 통신을 해서 요청되어진 응답물을 가지고 오게 된다. 이후 요청 받아진 결과는 라우터로 오게 되고 라우터는 누가 이 요청을 보냈는지 확인 한 후 이 요청을 보낸 기기로 결과물을 보내주게 되는 것입니다.
